# GUI 中间预览区重构设计

## 一、背景

当前桌面界面的中间区域同时承担了多种职责：

- 展示 TOP / FRONT-1 / FRONT-2 三个视角
- 展示预处理流程和表型提取流程
- 展示表型特征总览卡片
- 在不同状态下切换不同标题和不同图片来源

实际效果是：

- 初始状态与分析完成后的页面结构不一致
- 用户需要同时理解“视角”和“流程阶段”两套维度
- 分析完成后中间区域被大量拼图和特征卡片占满，视觉层级失控
- 切换样本、切换状态时，用户难以稳定理解当前正在看什么

本次重构只处理中间预览区。左侧数据目录 / 样本列表和右侧色卡设置 / 操作按钮 / 结果表 / 日志保留现状，不做布局级改动。

## 二、目标

本次重构的目标是把中间区域收敛成一个稳定、可预测的 4 图阶段预览面板。

具体目标如下：

- 中间区域始终只显示 4 张图
- 4 张图分别固定为：
  - 原始图像
  - 颜色校正
  - 背景消除
  - 表型提取
- 一组样本包含 3 个视角：`TOP`、`FRONT-1`、`FRONT-2`
- 页面中存在一个统一的当前视角状态
- 切换任意一张子图的左右翻页时，4 张图同步切换到同一视角
- 初始状态、预处理完成状态、表型提取完成状态使用同一套布局，不再在结构上跳变

## 三、不在本次范围内的内容

- 左侧目录与样本列表布局重构
- 右侧色卡选择、单株处理、批量处理、结果表、日志区域的重构
- 表型算法本身的调整
- 导出文件格式调整
- 全新的大图查看器
- 结果表字段增删

## 四、页面结构设计

### 1. 总体结构

中间区域保留在现有滚动区域中，但内部只保留两个层级：

1. 顶部状态与视角控制栏
2. 2 x 2 的四卡片图片区

不再在中间区展示“表型特征总览”卡片墙。

### 2. 顶部状态与视角控制栏

该区域用于表达唯一的全局上下文，包含：

- 当前样本编号
- 当前模式
  - 原始图像
  - 预处理结果
  - 表型提取结果
- 当前视角
  - `TOP`
  - `FRONT-1`
  - `FRONT-2`
- 当前视角索引显示
  - `1 / 3`
  - `2 / 3`
  - `3 / 3`

视角切换器采用统一控制器，而不是让 4 张卡片各自管理自己的视角状态。

推荐呈现方式：

- 左箭头
- 当前视角标签组
- 右箭头
- 当前页码

### 3. 四卡片区

四张卡片始终固定为：

1. `原始图像`
2. `颜色校正`
3. `背景消除`
4. `表型提取`

这四个标题在任何模式和任何视角下都不改变。

卡片只切换图片内容与简短状态说明，不切换卡片语义。

### 4. 方案选择结果

中间区采用已确认的方案 A：

- 顶部使用统一视角切换器
- 4 张卡片共用同一个当前视角状态
- 4 张卡片展示同一视角下的 4 个处理阶段

不采用“每张卡片各自带一套完整翻页头”的方案，以避免页面再次变乱。

## 五、四张卡片在不同状态下的显示规则

### 1. 初始状态

当用户刚选中样本、尚未执行预处理时：

- `原始图像`
  - 显示当前视角的原图
- `颜色校正`
  - 显示占位文案：`等待颜色校正`
- `背景消除`
  - 显示占位文案：`等待背景消除`
- `表型提取`
  - 显示占位文案：`等待表型提取`

### 2. 预处理完成后

当用户完成预处理但尚未完成表型提取时：

- `原始图像`
  - 显示当前视角的原图
- `颜色校正`
  - 显示当前视角的校正后图
  - 如果该视角校正失败，则显示原图并给出 `未校正` 标记
- `背景消除`
  - 显示占位文案：`等待表型提取`
- `表型提取`
  - 显示占位文案：`等待表型提取`

### 3. 表型提取完成后

当用户完成表型提取时：

- `原始图像`
  - 显示当前视角原图
- `颜色校正`
  - 显示当前视角校正后图
  - 若当前视角没有校正结果，回退显示原图，并显示 `未校正`
- `背景消除`
  - 显示当前视角对应的背景消除 / 分割结果图
- `表型提取`
  - 显示当前视角对应的最终测量覆盖图

### 4. 各视角的具体图片来源

#### TOP

- 原始图像：`group.top_image`
- 颜色校正：`PreprocessResult.calibrated_images["TOP"]`
- 背景消除：TOP 掩膜叠加图或 TOP segmentation overlay
- 表型提取：TOP 最终测量 overlay

#### FRONT-1

- 原始图像：`group.front_0_image`
- 颜色校正：`PreprocessResult.calibrated_images["FRONT-1"]`
- 背景消除：FRONT-1 掩膜叠加图或 segmentation overlay
- 表型提取：FRONT-1 最终测量 overlay

#### FRONT-2

- 原始图像：`group.front_180_image`
- 颜色校正：`PreprocessResult.calibrated_images["FRONT-2"]`
- 背景消除：FRONT-2 掩膜叠加图或 segmentation overlay
- 表型提取：FRONT-2 最终测量 overlay

#### 最终测量图的回退规则

如果某个视角暂时没有严格意义上的“最终测量 overlay”，则允许按以下优先级回退：

1. 该视角最终 trait overlay
2. 该视角最接近最终分析结果的 overlay
3. 该视角分割 overlay

即使发生回退，卡片标题仍保持 `表型提取`，但卡片底部状态文案要明确说明是 `使用近似结果图`。

## 六、交互设计

### 1. 统一视角状态

界面中新增一个统一的视角索引状态，例如：

- `0 -> TOP`
- `1 -> FRONT-1`
- `2 -> FRONT-2`

所有预览刷新都从这个状态派生，不允许每张卡片拥有自己的独立视角状态。

### 2. 顶部翻页器

顶部翻页器是主要入口：

- 点击左箭头：切到上一个可用视角
- 点击右箭头：切到下一个可用视角
- 当前视角标签同步更新
- 四张卡片同时更新

### 3. 卡片级翻页入口

为了满足“给每个子图加一个左右翻页查看功能”的要求，每张卡片仍提供轻量的左右翻页入口，但它不是独立状态机。

建议行为：

- 鼠标悬停卡片时，在图片左右两侧显示轻量箭头
- 点击任意卡片的箭头，本质上仍然调用统一的上一视角 / 下一视角逻辑
- 四张图同步切换

也就是说：

- 视觉上，每张图都“可以翻页”
- 逻辑上，仍然只有一个全局视角状态

### 4. 样本切换规则

切换样本时：

- 尽量保留当前视角
- 如果新样本包含该视角，则直接显示该视角的 4 张图
- 如果新样本缺少该视角，则自动切换到该样本第一个可用视角
- 状态栏显示：`当前视角不可用，已自动切换`

### 5. 模式切换规则

模式切换包括：

- 从原始状态进入预处理状态
- 从预处理状态进入表型提取状态
- 从已有结果恢复到表型提取状态

切换模式时：

- 保持当前视角不变
- 只刷新 4 张卡片的内容来源
- 不再无条件退回 TOP

## 七、视觉层级约束

为了避免页面重新变乱，界面需要遵守以下约束：

### 1. 标题稳定

以下内容固定不变：

- 中间区域标题
- 四张卡片标题

不再根据当前视角把卡片标题改成：

- `TOP 分割过程`
- `FRONT-1 分割过程`
- `FRONT-2 分割过程`

### 2. 信息减量

每张卡片底部只保留一行简短状态说明，例如：

- `已加载`
- `已校正`
- `未校正`
- `已分割`
- `已生成`
- `等待表型提取`
- `使用近似结果图`

不在卡片里塞长段说明。

### 3. 中间区不再展示表型总览卡片墙

当前 `trait_gallery_group` 造成的主要问题是：

- 与 4 张主卡片争抢注意力
- 引入第二套视觉网格
- 在分析完成后导致中间区域高度急剧膨胀

因此本次重构要求把它从中间区布局中拿掉。

表型数值仍以右侧结果表为主展示入口。

## 八、实现边界

### 1. 可复用现有控件

优先复用现有 4 个 `ImagePreviewCard`：

- `orig_preview`
- `calib_preview`
- `mask_preview`
- `final_preview`

不重新发明一套图片卡片控件。

### 2. 新增统一预览状态

界面层应新增一个统一的预览状态对象或等价字段组合，至少包含：

- 当前样本 ID
- 当前模式
- 当前视角
- 当前视角索引

后续所有中间区刷新都基于这一状态统一驱动。

### 3. 刷新逻辑收口

当前以下函数分别直接操作不同卡片，逻辑分散：

- `_show_original_preview_for_group()`
- `_show_preprocess_preview()`
- `_show_trait_preview_for_sample()`

本次重构要求把“卡片实际显示什么”收口到一个统一刷新函数中，例如：

- `_refresh_stage_previews()`

各状态函数只负责更新状态，不再分别直接拼图片来源。

### 4. 兼容现有逻辑

虽然 `trait_gallery_group` 在界面上不再使用，但第一阶段不要求彻底删除相关代码。

要求如下：

- 先把它从中间布局中移除或隐藏
- 保留必要的兼容代码，避免一次性牵连过多逻辑
- 后续如需要，再做二次清理

### 5. 批量翻页器保留

当前 `batch_pager_widget` 的职责是切换批量结果中的样本，不是切换视角。

本次重构中：

- 批量翻页器保留
- 顶部新翻页器负责视角切换
- 两者职责明确分开，不合并

## 九、数据来源与优先级

为了保证三种状态都能工作，图片来源优先级定义如下。

### 1. 原始图像

直接来自当前样本组的原始路径：

- TOP
- FRONT-1
- FRONT-2

### 2. 颜色校正图

优先从 `PreprocessResult.calibrated_images` 获取。

如果没有，再尝试从已有分析结果中的 calibration payload 获取。

如果仍然没有，则回退原图，并显示 `未校正`。

### 3. 背景消除图

优先使用内存中的 segmentation overlay。

如果内存中没有，但存在已导出的运行可视化目录，则允许从导出目录中选取对应视角的代表图片恢复展示。

### 4. 表型提取图

优先使用当前分析结果生成的最终 overlay。

如果当前内存中不存在，但样本已有历史运行可视化目录，则从最新一次导出目录恢复。

## 十、测试要求

### 1. GUI smoke tests

至少覆盖：

- 初始状态时 4 张卡片显示规则正确
- 预处理完成后，只有前两张卡片更新为有效图片
- 表型提取完成后，4 张卡片均根据当前视角刷新
- 任意卡片触发上一视角 / 下一视角时，4 张卡片同步切换
- 切换样本时保留当前视角
- 当前视角不可用时自动回退到可用视角

### 2. 兼容性验证

至少确认：

- 图片点击放大仍可用
- 批量样本翻页仍可用
- 右侧表型结果表不受影响
- 预处理和表型提取按钮行为不变

## 十一、验收标准

当满足以下条件时，本次重构视为完成：

- 中间区只显示 4 张固定阶段图
- 页面在初始、预处理后、分析后三种状态下结构一致
- 当前视角由统一控制器表达
- 任意一张卡片都可作为翻页入口
- 翻页时 4 张图同步切换
- 切换样本时不会无条件退回 TOP
- 中间区不再出现表型特征总览卡片墙
- 左侧和右侧布局保持现状

## 十二、实施建议

建议按以下顺序落地：

1. 先抽出统一的预览状态与刷新函数
2. 再接入顶部统一视角切换器
3. 再给每张卡片加轻量翻页入口
4. 最后移除中间区的 trait gallery 展示，并补 smoke tests

这样可以保证每一步都能验证，不会一次把中间区全部打碎。
